# 2026-07-19 — M26.1: stretch-tense engine (loop/20260719-m26-engine)

## What

First M26 iteration (M4 reactivated, unscored-first): pure engine support
for the two stretch constructions.

- **js/conjugator.js** — `STRETCH_TENSES = ["nearfuture", "progressive"]`
  + `STRETCH_LABELS`, deliberately SEPARATE from `TENSES` so nothing that
  iterates TENSES (question sampling, star math, informe, contrast) can
  ever pick them up (locked by a test). `gerund(verb)`: -ando/-iendo
  regulars from the infinitive; vowel-stem -yendo (leyendo, oyendo,
  trayendo — reuses `stemEndsInVowel`, so the seguir/qu digraph guard
  applies); **-ir verbs with a `pret3` flag reuse that same vowel change**
  (RAE: the gerund shares the third-person-preterite stem — pidiendo,
  siguiendo, sintiendo, sirviendo, consiguiendo, convirtiendo, muriendo
  all come free from existing dataset flags); `ger` override for the
  three the flags can't derive (diciendo, viniendo, pudiendo);
  ir → yendo. `conjugateStretch(verb, t)` builds the 6 phrase forms from
  hard-coded auxiliary paradigms ("voy a hablar", "estás pidiendo").
- **js/verbs.js** — `ger` flags on decir, poder, venir only.

## Verification

- tests/stretch.test.mjs (6 tests, all hand-verified against RAE):
  explicit gerunds for every formation class (regulars ×3 endings,

  verbs whose gerund is regular); full nearfuture + progressive
  paradigms; well-formedness sweep over ALL 100 verbs (no consonant+
  yendo, -ar never -iendo); STRETCH∉TENSES + TENSES.length===3 lock.
