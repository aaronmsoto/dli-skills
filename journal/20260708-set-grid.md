# 2026-07-08 — usability add-on: set-screen activity grid (owner-directed)

**Branch:** `loop/20260708-set-grid` · part of the M10 usability sprint

## What shipped

Section 2 of the group screen grew from five to six activity cards
(Práctica + Escucha) and the old `auto-fit/minmax(150px)` grid crammed
them into one row on desktop. Per owner direction: explicit
**3 columns on tablet/desktop** (2 rows of 3 with all six) and
**2 columns under 560px** (3 rows of 2). The single ⚔️ reto card keeps
its own full-width row via a `.contrast-row` override at both
breakpoints. Voiceless devices (5 cards) flow 3+2 / 2+2+1.

## Validation

`npm test` green · `npm run e2e` PASSED — new checks read the computed
`grid-template-columns` track count at 900px (3) and 360px (2), assert
the reto row stays single-track, and confirm no horizontal overflow at
360px. SPEC 4.2 updated; recorded as an owner add-on under M10.
