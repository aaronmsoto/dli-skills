# 2026-07-19 — M28.1: beacon Worker + D1, LIVE (loop/20260719-m28-server)

## What

First M28 iteration — the server side, built AND deployed in-session at
the owner's direction (token from the git-ignored .env; never printed):

- **server/worker.js** — vanilla-JS Worker, pure handlers exported for
  tests. `POST /beacon`: text/plain JSON (sendBeacon-simple, no
  preflight), strict page/event allowlists, invalid → silent 204;
  counters upsert per (UTC date, page, event). For `view` events it
  records TWO anonymous unique-device tokens (owner-approved uniques
  addendum): sha256(salt + ip + ua) truncated, with independent daily
  and monthly salts stored in D1 and DESTROYED on rotation — tokens are
  irreversible, unlinkable across windows, tied to no one. Raw IP/UA
  are transient in memory only. Per-IP rate limiting deliberately
  absent (storing IPs violates the amendment). `GET /aggregates`:
  public JSON (60-day counters, DAU, MAU + 400-day token sweep).
  `GET /`: self-contained owner dashboard page (lives on the Worker —
  the app stays pure static).
- **server/schema.sql** (counters, salts, uniques) + **wrangler.toml**
  (non-secret IDs only).
- **Provisioned live**: `conjuga-db` D1 created (ENAM), schema applied,
  Worker deployed → **https://conjuga-api.soto-c30.workers.dev**
  (Option A per the runbook; no DNS changes; Cloudflare stays out of
  the dliskills.com DNS path).
