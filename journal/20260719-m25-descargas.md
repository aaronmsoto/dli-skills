# 2026-07-19 — M25.3: ⬇️ Descargas download manager (loop/20260719-m25-descargas)

## What

Third M25 (PWA) iteration — the offline audio manager per SPEC §5.5:

- **js/descargas.js** (new module, in the SW shell): pure derivation +
  Cache API work. `clipUrlsForSet` builds a group's clip list in BOTH
  spoken shapes — person-prefixed ("yo hablo", tap-to-hear everywhere)
  AND bare forms ("hablo", the 🎧 Escucha prompt) — at both speeds, all
  six persons (vosotros filters the UI, never the data). Measured:
  326-350 unique clips ≈ 1.8-2.2 MB per group; ~39 MB all-in.
  `downloadGroup` = sequential fetch+`cache.put` into `audio-gNN` with
  skip-existing (resume for free); `deleteGroup`, `groupStatus`,
  `storageSnapshot` (`navigator.storage.estimate`), `requestPersistence`.
- **js/app.js** — `#/descargas` screen (standard chrome: crumbs + ☰ +
  footer; ☰ gains "⬇️ Descargas / Offline audio"): per-group rows with
  size estimate, progress bar, ⬇️ Descargar / ⬇️ Continuar / 🗑️ Borrar
  states, download-all (≈ 40 MB, reuses each row's skip-existing runner),
  storage readout, `persist()` on first download, iOS eviction warning,
  bilingual copy, aria-live announcements. Feature-detects `caches` and
  renders informative fallbacks (no manifest → "go online" note; no
  Cache API → "still streams" note).
- **js/audio.js** — exports `clipMap()` (the loaded manifest map).
- **sw.js** — shell list gains js/descargas.js; VERSION → shell-v2.

## Discoveries

- 🎧 Escucha plays BARE forms (app.js speak(t.answer)), not the
  person-prefixed texts — a group download without them would break the
  promised offline Escucha. Caught while reconciling my ~1 MB/group
  numbers against the plan's ~2 MB fact; sizes now measured from disk
  (6,832 B/clip average).
- `[hidden]` vs `.btn`: the class's `display` beats the UA hidden rule —
  same trap styles.css already fixed for `.menu-panel[hidden]`. Scoped
  `display:none` guards added for the dl controls.
